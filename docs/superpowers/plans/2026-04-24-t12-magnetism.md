**Goal:** When the agent calls `taskflow_update` / `taskflow_move` with a `task_id` the user did not explicitly address, refuse the mutation and return an `ambiguous_task_context` error naming the expected task. The guard only activates in the specific shape where the bot's immediately preceding message was a *short confirmation-question about exactly one task* — the shape responsible for the 2026-04-23 T12/T13 incident.

**Architecture:** Engine-level soft guard reading `/workspace/store/messages.db` (already mounted read-only per `src/container-runner.ts:195-203`). No MCP protocol change. No change to agent inputs. Wiring: `ipc-mcp-stdio.ts` passes `chatJid` into the engine's constructor options alongside the existing `triggerTurnId`.

**Tech Stack:** TypeScript (`taskflow-engine.ts`), `better-sqlite3` (both `taskflow.db` and `messages.db`), bash + inline JS for auditor, vitest tests.

## Design decisions (locked after Codex Q6 review)

1. **Single-ref short prior prompt enforcement** (not heuristic soup). Guard only fires when ALL of:
   - User message(s) for the current `triggerTurnId` have zero `T#`/`P#`/`SEC-` task references (via `agent_turn_messages` join using composite key `(message_id, message_chat_jid)` per Codex #1).
   - Bot message(s) immediately before the user turn, concatenated across consecutive messages within a 30-second window (per Codex split-message refinement), in the same `chat_jid`, together contain **exactly one** task reference.
   - That concatenated bot text matches a confirmation-question shape: contains `?` OR one of `{Cancelar, Mover, Atualizar, Reagendar, Concluir, Aprovar, Rejeitar, Remover, Arquivar, Fechar, Finalizar, Iniciar, Reabrir, Atribuir, Reatribuir}` (expanded per Codex CQ1) as a verb.
   - The agent's `task_id` parameter differs from the bot's single ref.

   **Fail-open on missing data** (per Codex race/WAL concern): if `agent_turn_messages` rows for the current turn are absent or `chatJid` / `messagesDb` / `triggerTurnId` are missing, guard does NOT fire. Emit a diagnostic counter `magnetism_fail_open_count` so we can see if the race condition is actually happening.

   Any other configuration (multi-ref bot msg, no prior bot, user msg has refs) → guard does NOT fire. Agent keeps full autonomy; magnetism risk is acknowledged in the deferred-items section.

2. **Soft guard, not hard reject.** Guard returns `{ success: false, error_code: 'ambiguous_task_context', expected_task_id, actual_task_id, bot_question_preview }`. Agent can retry with `confirmed_task_id: <actual>` to override (audit-logged). Template instructs the agent to ASK the user in the normal case, not silently override.

   **Correction per Codex #5:** `confirmed_task_id` is a new field and DOES require an MCP schema change — plan v2's "no MCP protocol change" was wrong. We add it to the zod schemas for `taskflow_update` and `taskflow_move` in `ipc-mcp-stdio.ts`. This is a ~10-line schema extension, not a breaking change (optional field).

3. **Guard runs AFTER `requireTask`, not at top of update/move.** Per Codex #6: `task_history` has no FK on `task_id`, so shadow-flag logging an invalid task_id would corrupt history. Placement: validate task_id exists via `requireTask`, THEN run `detectTaskIdMagnetism`.

3. **Shadow audit before live.** Detector in `auditor-script.sh` emits findings to a new JSON field. Kipp synthesizer ignores them for 14 days while we measure FP rate. Backfill 30 days of historical `messages.db` before enabling live shadow.

4. **Kill-switch env var `NANOCLAW_MAGNETISM_GUARD`**. Values: `off` (disabled entirely), `shadow` (detector only, guard disabled — default for rollout), `enforce` (guard active + detector live). Env var read at container start, passed into engine options.

5. **Revert SQL only when reconstructable.** `task_history.details` JSON contains `{from,to}` or `{changes:[old→new]}` for some actions but just `{changes:["Prazo removido"]}` for others. Kipp findings include `revert_sql` when old values exist, otherwise `revert: "not_reconstructable"`.

### Task 0.1 — Backfill analyzer script

**Files:** Create `/root/nanoclaw/scripts/magnetism-backfill.mjs`.

- [ ] **Step 1: Write the standalone analyzer**

```javascript
#!/usr/bin/env node
// Read-only analyzer: count task_id magnetism candidates in N days of history.
import Database from 'better-sqlite3';

const MESSAGES_DB = process.env.MESSAGES_DB || '/tmp/prod-snapshot-20260424/messages.db';
const TASKFLOW_DB = process.env.TASKFLOW_DB || '/tmp/prod-snapshot-20260424/taskflow.db';
const DAYS = Number(process.env.DAYS || 30);
const WINDOW_MS = 10 * 60 * 1000;

const TASK_REF_RE = /\b(?:[A-Z]{2,}-)?(?:T|P|M|R)\d+(?:\.\d+)*\b|\bSEC-[A-Z0-9]+(?:[.-][A-Z0-9]+)*\b/gi;
// Expanded verb list per Codex CQ1. Remover/Arquivar/Fechar/Finalizar cover verb-only
// prompts that don't contain "?" (e.g., "Confirma remover M4").
const CONFIRM_VERBS = /\b(Cancelar|Mover|Atualizar|Reagendar|Concluir|Aprovar|Rejeitar|Remover|Arquivar|Fechar|Finalizar|Iniciar|Reabrir|Atribuir|Reatribuir)\b/i;

function extractTaskRefs(text) {
  if (!text) return [];
  const matches = String(text).match(TASK_REF_RE);
  return matches ? Array.from(new Set(matches.map((m) => m.toUpperCase()))) : [];
}

function isConfirmationQuestion(text) {
  if (!text) return false;
  return text.includes('?') || CONFIRM_VERBS.test(text);
}

const msgDb = new Database(MESSAGES_DB, { readonly: true });
const tfDb = new Database(TASKFLOW_DB, { readonly: true });

const cutoff = new Date(Date.now() - DAYS * 86_400_000).toISOString();

// For every mutation in the window, check the shape.
const mutations = tfDb.prepare(
  `SELECT board_id, task_id, action, by, at, details FROM task_history
   WHERE action IN ('updated', 'moved', 'approve', 'conclude', 'cancel', 'reopen')
     AND at >= ? ORDER BY at ASC`,
).all(cutoff);

let totalMutations = 0, candidates = 0;
const byBoard = new Map();
const activeBoardDays = new Map(); // board_id -> Set of YYYY-MM-DD (for per-board-active-day rate)

for (const mut of mutations) {
  totalMutations += 1;
  // Track active boards
  const dayKey = String(mut.at).slice(0, 10);
  if (!activeBoardDays.has(mut.board_id)) activeBoardDays.set(mut.board_id, new Set());
  activeBoardDays.get(mut.board_id).add(dayKey);

  // Find the chat_jid for this mutation's board
  const board = tfDb.prepare(`SELECT group_jid FROM boards WHERE id = ?`).get(mut.board_id);
  if (!board?.group_jid) continue;
  const chatJid = board.group_jid;

  // Find the user message immediately before this mutation's timestamp.
  // NOTE: messages PK is (id, chat_jid) per Codex #1, so all message lookups scope by chat_jid.
  const mutMs = new Date(mut.at).getTime();
  const windowStartIso = new Date(mutMs - WINDOW_MS).toISOString();
  const userMsg = msgDb.prepare(
    `SELECT content, timestamp FROM messages
     WHERE chat_jid = ? AND is_from_me = 0 AND is_bot_message = 0
       AND timestamp >= ? AND timestamp <= ?
     ORDER BY timestamp DESC LIMIT 1`,
  ).get(chatJid, windowStartIso, mut.at);
  if (!userMsg) continue;
  if (extractTaskRefs(userMsg.content).length > 0) continue; // user named tasks explicitly

  // Find the bot message(s) immediately before the user message. Per Codex refinement:
  // concatenate consecutive bot messages within 30s to handle split "Cancelar T13?\nConfirme com sim."
  const BOT_CONCAT_WINDOW_MS = 30_000;
  const botConcatFloor = new Date(new Date(userMsg.timestamp).getTime() - BOT_CONCAT_WINDOW_MS).toISOString();
  const botMsgs = msgDb.prepare(
    `SELECT content, timestamp FROM messages
     WHERE chat_jid = ? AND (is_from_me = 1 OR is_bot_message = 1)
       AND timestamp < ? AND timestamp >= ?
     ORDER BY timestamp DESC LIMIT 3`,
  ).all(chatJid, userMsg.timestamp, botConcatFloor);
  if (botMsgs.length === 0) continue;
  const botContent = botMsgs.map((r) => r.content || '').reverse().join('\n');

  const botRefs = extractTaskRefs(botContent);
  if (botRefs.length !== 1) continue; // not single-ref (includes "0 refs" and ">=2 refs")
  if (!isConfirmationQuestion(botContent)) continue;

  if (botRefs[0].toUpperCase() === String(mut.task_id).toUpperCase()) continue; // agent correct

  candidates += 1;
  byBoard.set(mut.board_id, (byBoard.get(mut.board_id) ?? 0) + 1);
}

// Per-board rates scaled by active days, per Codex #10.
const perBoardRates = [];
for (const [boardId, count] of byBoard) {
  const activeDays = activeBoardDays.get(boardId)?.size ?? 0;
  perBoardRates.push({
    board_id: boardId,
    candidates: count,
    active_days: activeDays,
    weekly_normalized: activeDays > 0 ? (count / activeDays) * 7 : 0,
  });
}

console.log(JSON.stringify({
  days: DAYS,
  totalMutations,
  magnetismCandidates: candidates,
  projectedWeekly_fleet: (candidates / DAYS) * 7,
  perBoardRates: perBoardRates.sort((a, b) => b.weekly_normalized - a.weekly_normalized),
  // Gate: max per-board weekly-normalized rate. Target: max <= 1.
  max_per_board_weekly: Math.max(0, ...perBoardRates.map((b) => b.weekly_normalized)),
}, null, 2));
```

- [ ] **Step 2: Add the private helper method**

```typescript
private detectTaskIdMagnetism(
  params: { task_id: string; confirmed_task_id?: string },
): { shape: 'firing'; expected: string } | { shape: 'clear' } {
  if (params.confirmed_task_id) return { shape: 'clear' }; // explicit override
  if (this.guardMode === 'off') return { shape: 'clear' };
  if (!this.messagesDb || !this.chatJid || !this.triggerTurnId) return { shape: 'clear' }; // fail-open

  // Fetch the user message(s) for the current turn.
  // JOIN uses composite key (message_id, message_chat_jid) per Codex #1: messages PK
  // is (id, chat_jid), not id alone. Single-JID assumption may hold today, but the
  // schema doesn't guarantee it.
  const turnMessages = this.messagesDb.prepare(
    `SELECT m.content, m.timestamp FROM agent_turn_messages atm
     JOIN messages m ON m.id = atm.message_id AND m.chat_jid = atm.message_chat_jid
     WHERE atm.turn_id = ? ORDER BY atm.ordinal ASC`
  ).all(this.triggerTurnId) as Array<{ content: string; timestamp: string }>;
  if (turnMessages.length === 0) return { shape: 'clear' }; // fail-open on race/WAL

  const userText = turnMessages.map((r) => r.content || '').join(' ');
  if (this.extractRefs(userText).length > 0) return { shape: 'clear' }; // user named tasks

  const firstTurnTs = turnMessages[0].timestamp;
  if (!firstTurnTs) return { shape: 'clear' };

  // Concatenate bot messages within 30s before the turn started (handles split
  // "Cancelar T13?" + "Confirme com sim" per Codex split-message refinement).
  const BOT_CONCAT_WINDOW_MS = 30_000;
  const botFloor = new Date(new Date(firstTurnTs).getTime() - BOT_CONCAT_WINDOW_MS).toISOString();
  const priorBots = this.messagesDb.prepare(
    `SELECT content FROM messages
     WHERE chat_jid = ? AND (is_from_me = 1 OR is_bot_message = 1)
       AND timestamp < ? AND timestamp >= ?
     ORDER BY timestamp DESC LIMIT 3`
  ).all(this.chatJid, firstTurnTs, botFloor) as Array<{ content: string }>;
  if (priorBots.length === 0) return { shape: 'clear' };

  const botText = priorBots.map((r) => r.content || '').reverse().join('\n');
  const botRefs = this.extractRefs(botText);
  if (botRefs.length !== 1) return { shape: 'clear' }; // digest/list or no ref
  if (!CONFIRM_VERBS.test(botText) && !botText.includes('?')) return { shape: 'clear' };
  if (botRefs[0].toUpperCase() === params.task_id.toUpperCase()) return { shape: 'clear' };

  return { shape: 'firing', expected: botRefs[0] };
}

private extractRefs(text: string): string[] {
  if (!text) return [];
  const matches = text.match(TASK_REF_RE);
  return matches ? Array.from(new Set(matches.map((m) => m.toUpperCase()))) : [];
}
```

  try {
    const details = JSON.parse(historyRow.details || '{}');

    }
    // Other shapes NOT reliably reconstructable.
    return null;
  } catch {
    return null;
  }
}
